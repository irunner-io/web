Air-Gap & Licensing

iRunner operates fully disconnected from iRunner's servers. There is no phone-home requirement. Licensing is enforced locally via Ed25519 signed keys that are validated entirely on your infrastructure.

No phone-home

iRunner never contacts any external licensing server. The control plane validates the license key locally at startup and enforces limits without any network call. This means:

  • Full air-gap support — works in isolated VPCs with no internet access (apart from GitHub API for runner registration)
  • No telemetry, analytics, or usage reporting sent externally
  • No dependency on iRunner's infrastructure for operation
  • License validation works even if DNS is restricted

License structure

The license key is a signed JWT containing claims about your entitlements:

ClaimDescriptionExample
customerCustomer organization nameAcme Corp
planLicense tierteam, enterprise
expExpiration date (Unix timestamp)1735689600
max_concurrencyMaximum concurrent runners50
featuresEnabled feature flags["cache", "docker_cache", "warm_pools"]

License validation

The license is validated using Ed25519 public-key cryptography:

  1. iRunner's build embeds the Ed25519 public key
  2. At startup, the control plane reads the license key from config or environment
  3. The JWT signature is verified against the embedded public key
  4. Claims are checked: expiration, concurrency limits, feature flags
  5. If validation passes, the system starts normally
  6. If validation fails (expired, invalid signature), the system logs the error and refuses to start
Note

The Ed25519 signature cannot be forged without the private key (held only by iRunner). Modifying any claim in the JWT invalidates the signature.

Trial mode

iRunner includes a built-in trial that requires no license key:

LimitValue
Duration14 days from first job
Total jobs500
Max concurrency5 runners
FeaturesAll features enabled

No registration, credit card, or contact with iRunner is required to start the trial. Deploy the stack and run jobs immediately.

Providing the license key

The license key is a base64-encoded string. Provide it via environment variable or config.yaml:

Option 1: Environment variable

Terminal
# Store in AWS Secrets Manager (recommended)
aws secretsmanager create-secret \
  --name ir-west/license-key \
  --secret-string "eyJhbGciOiJFZDI1NTE5Ii..."

# The CloudFormation stack reads this secret automatically

Option 2: config.yaml

config.yaml
license:
  key: "eyJhbGciOiJFZDI1NTE5IiwidHlwIjoiSldUIn0.eyJjdXN0b21lciI6IkFjbWUgQ29ycCIsInBsYW4iOiJlbnRlcnByaXNlIiwiZXhwIjoxNzM1Njg5NjAwLCJtYXhfY29uY3VycmVuY3kiOjUwLCJmZWF0dXJlcyI6WyJjYWNoZSIsImRvY2tlcl9jYWNoZSIsIndhcm1fcG9vbHMiXX0.SIGNATURE"

Enforcement behavior

When license limits are reached:

ConditionBehavior
License expiredControl plane refuses to start; logs error
Max concurrency reachedNew jobs are queued until a runner becomes available
Trial job limit reachedNew jobs are rejected with an error; existing jobs complete
Feature not licensedFeature-specific config sections are ignored

License renewal

When your license approaches expiration:

  1. The control plane logs warnings starting 30 days before expiration
  2. A metric ir_license_days_remaining is emitted for alerting
  3. To renew: update the secret in Secrets Manager with the new key and restart the control plane (or wait for the next config reload)
Tip

Set up an alert on ir_license_days_remaining < 14 to get notified before your license expires.
