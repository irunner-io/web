ARM64 / Graviton

Run CI jobs on AWS Graviton (ARM64) processors for up to 20% better price-performance compared to equivalent x64 instances. Same workflow syntax — just a different label.

Why ARM64?

AWS Graviton processors offer several advantages for CI workloads:

  • Cost: ~20% cheaper than equivalent x64 instances
  • Performance: Competitive or better single-threaded performance for compiled languages
  • Native ARM builds: Build ARM Docker images and binaries without emulation
  • Production parity: If you deploy to Graviton in production, test on Graviton in CI

Configuration

Set arch: arm64 in your pool definition and use a Graviton instance type:

config.yaml
pools:
  - name: linux-arm64
    platform: ec2
    arch: arm64
    instance_type: m7g.2xlarge      # Graviton3
    ami: ami-0arm64-runner-base     # ARM64-specific AMI
    disk_size_gb: 100
    subnet_ids:
      - subnet-aaa111
      - subnet-bbb222
    security_group_ids:
      - sg-runners-001
    spot_enabled: true

  - name: linux-arm64-large
    platform: ec2
    arch: arm64
    instance_type: c7g.4xlarge      # Graviton3 compute-optimized
    ami: ami-0arm64-runner-base
    disk_size_gb: 200
    subnet_ids:
      - subnet-aaa111
      - subnet-bbb222
    security_group_ids:
      - sg-runners-001

label_mappings:
  ir-arm64: linux-arm64
  ir-arm64-large: linux-arm64-large

Graviton instance types

Common Graviton instance types for CI workloads:

Instance typevCPUMemoryUse case

Workflow usage

Using ARM64 runners is as simple as changing the label:

.github/workflows/ci.yml
jobs:
  test-x64:
    runs-on: ir-large
    steps:
      - uses: actions/checkout@v4
      - run: make test

  test-arm64:
    runs-on: ir-arm64
    steps:
      - uses: actions/checkout@v4
      - run: make test

  # Build multi-arch Docker image
  docker-multiarch:
    runs-on: ir-arm64
    steps:
      - uses: actions/checkout@v4
      - uses: docker/setup-buildx-action@v3
      - uses: docker/build-push-action@v5
        with:
          context: .
          platforms: linux/arm64
          push: true
          tags: myapp:latest-arm64

Important considerations

ARM-specific AMI required

You cannot use an x64 AMI on a Graviton instance (and vice versa). The ami field must reference an ARM64-compatible AMI. iRunner publishes separate base AMIs for each architecture.

Software compatibility

Most CI tools support ARM64 natively:

  • Go, Rust, Java, Python, Node.js — all have native ARM64 support
  • Docker — runs natively on ARM64
  • GitHub Actions runner — official ARM64 builds available

However, some tools may not have ARM64 binaries. Check your toolchain before switching.

ECS Fargate ARM64

Fargate also supports ARM64 via the ARM64 runtime platform:

config.yaml
pools:
  - name: fargate-arm64
    platform: ecs
    arch: arm64
    cpu: 4096
    memory: 8192
    image: "123456789012.dkr.ecr.us-west-2.amazonaws.com/runner:latest-arm64"

label_mappings:
  ir-ecs-arm64: fargate-arm64
Note

When using ARM64 with ECS Fargate, ensure your container image is built for the linux/arm64 platform. Multi-arch images (built with docker buildx) work seamlessly.
