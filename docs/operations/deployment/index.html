Deployment

iRunner deploys entirely via AWS CloudFormation. A single stack provisions all infrastructure, and the control plane runs as an ECS service with configuration stored in S3.

What the stack creates

The CloudFormation template provisions the following resources:

  • VPC with configurable CIDR block
  • Subnets — public subnets for the ALB, private subnets dedicated per compute tier (EC2, ECS, EKS)
  • Application Load Balancer with HTTPS listener and health checks
  • ECS cluster + service running the control plane as a Fargate task
  • SQS queues (.fifo) for job dispatch and lifecycle events
  • DynamoDB tables for pool state, job tracking, and instance metadata
  • S3 buckets for releases, config, and cache storage
  • IAM roles for the control plane task, runner instances, and cross-service access
  • Cloud Map namespace for internal service discovery

Key parameters

ParameterDescriptionExample
PrefixResource naming prefix (must be unique per account)ir-west
VpcCidrCIDR block for the VPC10.0.0.0/16
GitHubAppIdGitHub App ID for webhook authentication123456
GitHubInstallationIdInstallation ID from the GitHub App install789012

Deploy the stack

Terminal
aws cloudformation create-stack \
  --stack-name ir-controlplane \
  --template-url https://ir-releases.s3.amazonaws.com/cloudformation/latest.yaml \
  --capabilities CAPABILITY_IAM CAPABILITY_NAMED_IAM \
  --parameters \
    ParameterKey=Prefix,ParameterValue=ir-west \
    ParameterKey=VpcCidr,ParameterValue=10.0.0.0/16 \
    ParameterKey=GitHubAppId,ParameterValue=123456 \
    ParameterKey=GitHubInstallationId,ParameterValue=789012

# Wait for completion
aws cloudformation wait stack-create-complete --stack-name ir-controlplane

Stack outputs

Once the stack is complete, retrieve the outputs you will need for subsequent steps:

Terminal
aws cloudformation describe-stacks --stack-name ir-controlplane \
  --query "Stacks[0].Outputs" --output table
OutputDescription
WebhookEndpointPublic ALB URL for GitHub webhook delivery
InternalEndpointPrivate Cloud Map endpoint for VPC-internal access
ReleasesBucketNameS3 bucket for config and binary releases
EcsClusterArnARN of the ECS cluster running the control plane
EcsServiceNameName of the ECS service

Generate configuration

Use the config generator script to produce a config.yaml tailored to your stack:

Terminal
./deploy/generate-config.sh \
  --stack-name ir-controlplane \
  --github-app-id 123456 \
  --github-installation-id 789012 \
  > config.yaml

Upload configuration to S3

The control plane reads its configuration from a well-known S3 path:

Terminal
aws s3 cp config.yaml s3://{prefix}-releases-{account-id}/config/config.yaml

Build and push the Docker image

Terminal
# Authenticate with ECR
aws ecr get-login-password --region us-west-2 | \
  docker login --username AWS --password-stdin {account-id}.dkr.ecr.us-west-2.amazonaws.com

# Build the image
docker build -t ir-control-plane:latest .

# Tag and push
docker tag ir-control-plane:latest \
  {account-id}.dkr.ecr.us-west-2.amazonaws.com/ir-control-plane:latest
docker push {account-id}.dkr.ecr.us-west-2.amazonaws.com/ir-control-plane:latest

Register task definition and update ECS service

Terminal
# Register a new task definition revision
aws ecs register-task-definition \
  --cli-input-json file://deploy/task-definition.json

# Update the ECS service to use the new task definition
aws ecs update-service \
  --cluster {prefix}-cluster \
  --service {prefix}-control-plane \
  --task-definition ir-control-plane \
  --force-new-deployment
Tip

The ECS service performs a rolling deployment by default. The ALB health check ensures traffic only routes to healthy tasks before draining old ones.
