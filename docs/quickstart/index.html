Quickstart

Deploy iRunner and run your first build in 15 minutes. This guide walks you through deploying the CloudFormation stack, installing the GitHub App, and triggering your first job.

Prerequisites

  • An AWS account with permission to create CloudFormation stacks
  • A GitHub organization (or personal account) with admin access
  • AWS CLI v2 installed and configured

Step 1: Deploy the CloudFormation stack

The stack creates all required infrastructure: VPC, subnets, ALB, ECS control plane, SQS queues, DynamoDB tables, S3 buckets, and IAM roles.

Terminal
aws cloudformation create-stack \
  --stack-name ir-controlplane \
  --template-url https://ir-releases.s3.amazonaws.com/cloudformation/latest.yaml \
  --capabilities CAPABILITY_IAM CAPABILITY_NAMED_IAM \
  --parameters \
    ParameterKey=Prefix,ParameterValue=ir-west \
    ParameterKey=VpcCidr,ParameterValue=10.0.0.0/16

Wait for the stack to complete (~5 minutes):

Terminal
aws cloudformation wait stack-create-complete --stack-name ir-controlplane

Step 2: Create a GitHub App

iRunner uses a private GitHub App for secure webhook delivery and runner registration.

  1. Go to your GitHub organization → Settings → Developer settings → GitHub Apps → New GitHub App
  2. Configure the app:
    • Name: iRunner (your-org)
    • Webhook URL: the WebhookEndpoint stack output
    • Webhook secret: generate a strong random string
  3. Set permissions:
    ScopePermissionAccess
    RepositoryActionsRead
    RepositoryAdministrationRead & Write
    RepositoryMetadataRead
    OrganizationSelf-hosted runnersRead & Write
  4. Subscribe to the workflow_job event
  5. Create the app, then install it on your organization

Step 3: Store secrets

Store the GitHub App private key and webhook secret in AWS Secrets Manager:

Terminal
# Store the private key
aws secretsmanager create-secret \
  --name ir-west/github-app-private-key \
  --secret-string file://private-key.pem

# Store the webhook secret
aws secretsmanager create-secret \
  --name ir-west/webhook-secret \
  --secret-string "your-webhook-secret"

Step 4: Generate and upload config

Use the config generator to create your config.yaml:

Terminal
./deploy/generate-config.sh \
  --stack-name ir-controlplane \
  --github-app-id 123456 \
  --github-installation-id 789012 \
  | aws s3 cp - s3://ir-west-releases-ACCOUNT_ID/config/config.yaml

Step 5: Update your workflow

Change the runs-on label in your workflow file:

.github/workflows/ci.yml
jobs:
  build:
    runs-on: ir-cold    # any configured label
    steps:
      - uses: actions/checkout@v4
      - run: make test

Step 6: Trigger a build

Terminal
git commit --allow-empty -m "test: trigger iRunner"
git push

Watch the GitHub Actions UI — the job should be picked up within seconds.

Verify the deployment

Check control plane status from within your VPC:

Terminal
curl -s http://control-plane.ir-west.internal:8080/internal/status | jq .
Tip

The control plane's internal endpoint is only accessible from within the VPC. Use a bastion host or SSM Session Manager to reach it.
