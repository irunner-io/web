Changelog

Version history for iRunner. All notable changes are documented here in reverse chronological order.

v0.5.4 — 2026-06-10

Added

  • License enforcement with signed JWT validation at job submission time.
  • ALB lockdown: default deny on all unrecognized paths (returns 403).
  • Internal DNS routing via Route 53 private hosted zones for control plane discovery.
  • Cache multipart upload support for artifacts larger than 100 MB.

Changed

  • Control plane health check moved from /healthz to /health for ALB compatibility.
  • License expiry now checked every 60 seconds instead of only at startup.

Fixed

  • Race condition in pool capacity counter when multiple jobs start simultaneously.
  • Cache upload timeout for large monorepo artifacts (increased to 10 minutes).

v0.5.3 — 2026-05-20

Added

  • Docker layer cache backed by S3 and ECR for fast image rebuilds.
  • ECS Fargate Spot support via ecs.spot_enabled pool configuration.
  • Prometheus metrics for Docker cache hit/miss rate.

Changed

  • ECS tasks now use ephemeral storage configuration (previously hardcoded at 21 GB).

Fixed

  • ECS task role assumption failure when cache service was enabled.
  • Fargate Spot interruption not triggering graceful job cancellation.

v0.5.2 — 2026-04-28

Added

  • Label patterns with regex matching and optional constraints (max_concurrency, allowed_repos).
  • ARM64 / Graviton pool support with arch: arm64 configuration.
  • Per-job cost tracking exposed via /internal/jobs endpoint.

Changed

  • Label routing now evaluates label_patterns after label_mappings (static mappings take priority).
  • AMI lookup supports both ami-* IDs and SSM parameter paths.

Fixed

  • ARM64 instances failing to register with GitHub due to incorrect runner binary architecture.
  • Constraint violation not returning a clear error to the GitHub check run.

v0.5.1 — 2026-04-05

Added

  • S3 cache service with presigned URL upload/download for build artifacts.
  • Warm pool housekeeping: periodic reconciliation of idle instances against min_warm.
  • SQS-based spot termination handler for graceful shutdown.

Changed

  • Instance boot time reduced by 8 seconds via parallel package installation.
  • DynamoDB capacity switched to on-demand (pay-per-request) mode.

Fixed

  • Warm pool instances not being reused when jobs arrive within 2 seconds of launch.
  • S3 presigned URL expiry too short for large cache uploads (increased presign_ttl default to 15m).

v0.5.0 — 2026-03-15

Added

  • Initial release of iRunner.
  • EC2 compute backend with on-demand and spot instance support.
  • GitHub App webhook integration for workflow_job events.
  • Basic pool management with min_warm and max_total controls.
  • CloudFormation stack for one-click infrastructure deployment.
  • Ephemeral runners: each job gets a fresh instance, terminated after completion.
  • SQS-based job queue for reliable event processing.
  • DynamoDB state tracking for jobs and pool capacity.
