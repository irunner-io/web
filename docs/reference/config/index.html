Docs/Reference/config.yaml

config.yaml

Complete reference for iRunner's configuration file. The control plane reads this from S3 on startup and watches for changes. All fields are documented with their defaults and constraints.

server

Control plane server settings.

FieldTypeDefaultDescription
external_urlstring—Internal DNS endpoint for the control plane (e.g. http://control-plane.ir-west.internal:8080). Used by runners to call back.
portinteger8080Port the control plane listens on.

github

GitHub App credentials and configuration.

FieldTypeDescription
app_idintegerGitHub App ID from the app settings page.
installation_idintegerInstallation ID for the target organization.
private_key_pathstringAWS Secrets Manager ARN containing the GitHub App private key PEM.
webhook_secret_pathstringAWS Secrets Manager ARN containing the webhook HMAC secret.
app_config_arnstringSecrets Manager ARN for the full GitHub App configuration JSON (optional, used for automated installs).

aws

AWS resource references created by the CloudFormation stack.

FieldTypeDescription
regionstringAWS region (e.g. us-west-2).
sqs.jobs_queue_urlstringSQS queue URL for incoming workflow_job events.
sqs.termination_queue_urlstringSQS queue URL for EC2 spot termination notices.
sqs.events_queue_urlstringSQS queue URL for internal lifecycle events.
dynamodb.jobs_tablestringDynamoDB table for job state tracking.
dynamodb.pools_tablestringDynamoDB table for pool capacity state.
dynamodb.locks_tablestringDynamoDB table for distributed locking.
s3.releases_bucketstringS3 bucket containing agent binaries and config.
s3.agent_s3_keystringS3 key for the runner agent binary.
s3.presign_ttldurationTTL for presigned S3 URLs (e.g. 15m).

cache

S3 build cache and Docker layer cache settings.

FieldTypeDefaultDescription
enabledboolfalseEnable the S3 cache service.
bucketstring—S3 bucket for build cache artifacts.
ttl_daysinteger7Days before cache entries expire.
docker_bucketstring—S3 bucket for Docker layer cache (can be same as bucket).
docker_registrystring—ECR registry URL for cached Docker images.
docker_registry_ttl_daysinteger14Days before Docker layer cache entries expire.

default_pool

The pool used when a runs-on label doesn't match any label_mappings or label_patterns. Uses the same fields as entries under pools.

pools

Named pool definitions. Each pool configures a compute backend.

EC2 pool fields

FieldTypeDefaultDescription
instance_typestring—EC2 instance type (e.g. m7i.2xlarge).
amistring—AMI ID for the runner instance.
disk_size_gbinteger50Root EBS volume size in GB.
min_warminteger0Minimum number of pre-warmed idle instances.
max_totalinteger10Maximum total instances (warm + active).
spot_enabledboolfalseUse EC2 Spot Instances for this pool.
osstringlinuxOperating system (linux or windows).
archstringamd64CPU architecture (amd64 or arm64).
subnets[]string—Subnet IDs for instance placement.
security_groups[]string—Security group IDs attached to instances.
instance_profile_arnstring—IAM instance profile ARN for the runner.
sudoers_extra[]string—Additional sudoers entries for the runner user.
metrics_collectorstring—Metrics agent to install (cloudwatch or prometheus).
packages[]string—Extra packages to install at boot.

ECS pool fields

Set driver: ecs to use ECS Fargate as the compute backend.

FieldTypeDefaultDescription
driverstringec2Set to ecs for Fargate pools.
ecs.cluster_arnstring—ECS cluster ARN to run tasks in.
ecs.imagestring—Container image for the runner task.
ecs.execution_role_arnstring—IAM role for ECS task execution (image pull, logs).
ecs.task_role_arnstring—IAM role assumed by the running task.
ecs.sizestring—Fargate size spec (e.g. 2vcpu-4gb).
ecs.spot_enabledboolfalseUse Fargate Spot capacity.
ecs.subnets[]string—Subnet IDs for task placement.
ecs.security_groups[]string—Security group IDs for the task ENI.
ecs.ephemeral_storage_gbinteger21Ephemeral storage in GB (21–200).

label_mappings

Static map of runs-on label strings to pool names. Simple and fast lookup.

config.yaml
label_mappings:
  ir-small:
    pool: small-ec2
  ir-large:
    pool: large-ec2
  ir-fargate:
    pool: ecs-default

label_patterns

Regex-based label routing with optional constraints. Evaluated in order; first match wins.

FieldTypeDescription
patternstringRegex pattern matched against the runs-on label.
poolstringPool name to route matching jobs to.
constraintsobjectOptional constraints: max_concurrency, allowed_repos.
config.yaml
label_patterns:
  - pattern: "^ir-gpu-.*"
    pool: gpu-pool
    constraints:
      max_concurrency: 4
      allowed_repos:
        - "org/ml-training"
  - pattern: "^ir-arm-.*"
    pool: arm64-pool

config.yaml
server:
  external_url: http://control-plane.ir-west.internal:8080
  port: 8080

github:
  app_id: 123456
  installation_id: 789012
  private_key_path: arn:aws:secretsmanager:us-west-2:111111111111:secret:ir-west/github-app-private-key
  webhook_secret_path: arn:aws:secretsmanager:us-west-2:111111111111:secret:ir-west/webhook-secret

aws:
  region: us-west-2
  sqs:
    jobs_queue_url: https://sqs.us-west-2.amazonaws.com/111111111111/ir-west-jobs
    termination_queue_url: https://sqs.us-west-2.amazonaws.com/111111111111/ir-west-termination
    events_queue_url: https://sqs.us-west-2.amazonaws.com/111111111111/ir-west-events
  dynamodb:
    jobs_table: ir-west-jobs
    pools_table: ir-west-pools
    locks_table: ir-west-locks
  s3:
    releases_bucket: ir-west-releases-111111111111
    agent_s3_key: agent/ir-agent-linux-amd64
    presign_ttl: 15m

cache:
  enabled: true
  bucket: ir-west-cache-111111111111
  ttl_days: 7
  docker_bucket: ir-west-docker-cache-111111111111
  docker_registry: 111111111111.dkr.ecr.us-west-2.amazonaws.com/ir-cache
  docker_registry_ttl_days: 14

default_pool:
  instance_type: m7i.large
  ami: ami-0abcdef1234567890
  disk_size_gb: 50
  min_warm: 1
  max_total: 10
  spot_enabled: false
  os: linux
  arch: amd64
  subnets:
    - subnet-aaa111
    - subnet-bbb222
  security_groups:
    - sg-ccc333
  instance_profile_arn: arn:aws:iam::111111111111:instance-profile/ir-west-runner

pools:
  large-ec2:
    instance_type: m7i.2xlarge
    ami: ami-0abcdef1234567890
    disk_size_gb: 100
    min_warm: 2
    max_total: 20
    spot_enabled: true
    os: linux
    arch: amd64
    subnets:
      - subnet-aaa111
      - subnet-bbb222
    security_groups:
      - sg-ccc333
    instance_profile_arn: arn:aws:iam::111111111111:instance-profile/ir-west-runner
    packages:
      - docker-ce
      - docker-buildx-plugin

  arm64-pool:
    instance_type: m7g.xlarge
    ami: ami-0arm64example
    disk_size_gb: 50
    min_warm: 1
    max_total: 8
    spot_enabled: true
    os: linux
    arch: arm64
    subnets:
      - subnet-aaa111
    security_groups:
      - sg-ccc333
    instance_profile_arn: arn:aws:iam::111111111111:instance-profile/ir-west-runner

  ecs-default:
    driver: ecs
    ecs:
      cluster_arn: arn:aws:ecs:us-west-2:111111111111:cluster/ir-west-runners
      image: 111111111111.dkr.ecr.us-west-2.amazonaws.com/ir-runner:latest
      execution_role_arn: arn:aws:iam::111111111111:role/ir-west-ecs-execution
      task_role_arn: arn:aws:iam::111111111111:role/ir-west-ecs-task
      size: 2vcpu-4gb
      spot_enabled: true
      subnets:
        - subnet-ddd444
      security_groups:
        - sg-eee555
      ephemeral_storage_gb: 50

label_mappings:
  ir-small:
    pool: default
  ir-large:
    pool: large-ec2
  ir-arm:
    pool: arm64-pool
  ir-fargate:
    pool: ecs-default

label_patterns:
  - pattern: "^ir-gpu-.*"
    pool: gpu-pool
    constraints:
      max_concurrency: 4
      allowed_repos:
        - "org/ml-training"
  - pattern: "^ir-arm-.*"
    pool: arm64-pool
